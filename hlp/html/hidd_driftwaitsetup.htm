Wait for Drift Setup Dialog

This dialog allows you to set parameters for the Wait for Drift task, which measures drift repeatedly until it reaches a criterion level or a time limit is reached.  These parameters are used by the DriftWaitTask script command unless they are overridden by non-zero entries to the command.

Images to measure drift between

Use these radio buttons to select what kind of images to use for measuring drift.  The choices are:

  • Between successive Trial images, possibly with a modified exposure time and binning.
  • Between successive Focus images, using the Focus parameters.
  • Between the last images of successive autofocus runs.  This choice allows focus to be refined at the same time as drift is measured, while still giving an accurate measure of low drift rates because the time between correlated images is as long as you specify.
  • Within individual autofocus runs, using the drift estimated by the shift between the first and the third image.  This method is suitable only for measuring higher drift rates, unless the time between those images is relatively long.  It is the only method that can finish in less than the time between successive measurements.

For both of the autofocus options, the program will take 3 Focus images for 'drift protection' regardless of the setting of the option for this in the Focus menu.

Change Trial parameters

Select this option to use a specified exposure time and binning whenever using Trial images.  Use the text box to set the Trial exposure and the spin button to set the Binning.

Time between drift measurements

Enter the time between successive measurements in the text box.  When measuring drift within autofocus runs, this is the interval between the start of the autofocus runs; otherwise, it is the time over which drift is measured.

Drift rate to reach

Enter the criterion (target) drift rate in the text box, in the units show after the box (nm versus Angstroms).

Maximum time to wait

Enter the maximum total time to wait for the drift to reach the target rate.

Generate error if target drift rate is not reached

Select this option to have the program put up a message box and issue a general stop command to anything else running when the target drift rate is not achieved after the maximum wait time has elapsed.  With the option off, the program will print a failure message to the log and scripts can test for success with the 'GetLastTaskDrift' command.

Correct drift with image shift

Select this option to have image shift applied to correct for the drift between each pair of successive images; this option is not available when measuring within autofocuses.

Show Angstroms/sec instead of nm/sec

This option is here because nm/sec, used elsewhere in the program, becomes awkward at low drift rates.
